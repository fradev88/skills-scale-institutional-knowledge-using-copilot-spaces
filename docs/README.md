# OctoAcme Project Management Docs

Welcome to the OctoAcme Project Management Documentation set. This folder contains comprehensive guides and processes for managing projects at OctoAcme.

## Overview of OctoAcme Project Management Processes

OctoAcme employs an iterative, stakeholder-aligned, customer-value-focused approach to project management. Our methodology is built on five core principles and five key phases that guide how we plan, execute, and deliver projects.

### Core Principles
- **Customer-First**: Prioritize customer value and usability in all decisions
- **Iterative Delivery**: Deliver small, testable increments to gather feedback early
- **Clear Ownership**: Each project has named Project Manager (PM) and Product Lead roles
- **Data-Informed Decisions**: Measure impact and iterate based on evidence
- **Psychological Safety**: Encourage feedback, learning, and continuous improvement

### Project Management Lifecycle

Our projects progress through five essential phases:

1. **Initiation**: Define the problem statement, identify stakeholders, and establish a high-level timeline
2. **Planning**: Detail the scope, allocate resources, establish milestones, and identify dependencies
3. **Execution**: Build and test features, collaborate on design, and iterate based on feedback
4. **Release**: Deploy changes, verify functionality, and communicate outcomes to stakeholders
5. **Retrospective & Continuous Improvement**: Capture learnings, document lessons, and identify improvements

### Key Roles

- **Project Manager (PM)**: Coordinates delivery, manages schedules, monitors risks, and ensures communication
- **Product Manager (PdM)**: Defines outcomes, prioritizes the backlog, and measures success
- **Development Team**: Implements features, collaborates on design, and ensures code quality
- **QA/Testing**: Validates quality and confirms acceptance criteria are met
- **Stakeholders**: Provide strategic inputs and approvals

### Communication Cadence

- Weekly syncs between PM and Product Manager
- Twice-weekly standups for the delivery team
- Monthly stakeholder updates
- Ad-hoc escalations as needed
